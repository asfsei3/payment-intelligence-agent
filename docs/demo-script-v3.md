## Act I — 課題説明(0:00 - 0:45 / 45秒 / 5スライド)

> **画面表現**: ブランドカラー(forest green + champagne gold)の静止スライド。本番アプリは映さない。スライド遷移は 2 段階(旧スライドを完全に消す → 0.4 秒待つ → 新スライド表示)で重なり残留を防止。

### Scene 1 — タイトル(0:00 - 0:08 / 8秒)

- **画面**: 深緑背景。中央に大きく作品名。
- **メインコピー**: `Payment Intelligence Agent`
- **サブ**: `決済エラー対応を、AIエージェントで整理する Revenue Ops Desk`
- **上に小さく**: `Microsoft Agent Hackathon 2026 応募作品`

### Scene 2 — 問題提起(0:08 - 0:18 / 10秒)

- **画面**: シンプルな図解。サブスク契約者100人のアイコンが並び、そのうち数人が赤くマークされ「決済失敗」と注記。
- **見出し**: `サブスクでは、毎月一定数の決済失敗が発生する`
- **本文**:
  > カードの **期限切れ・残高不足・本人認証エラー** などにより、請求が完了しない取引が発生します。
  > これは例外ではなく、継続課金では **避けにくい運用課題** です。

### Scene 3 — その先に何が起きるか(0:18 - 0:27 / 9秒)

- **画面**: 矢印フロー図「決済失敗 → 放置 → 売上が消える」。
- **見出し**: `失敗を放置すると、請求できなかった売上が積み上がる`
- **本文**:
  > 失敗した取引は「請求できなかった売上」として消えます。
  > **気づかないうちに毎月積み上がる** — これが今回解決したい問題です。

### Scene 4 — 現場で何が起きているか(0:27 - 0:39 / 12秒)

- **画面**: 4象限のグリッド。①決済会社の管理画面、②CSVファイル、③カスタマーサポートがメール、④経営報告書、それぞれ別の担当者アイコン付き。
- **見出し**: `対応は、4つのツールと担当者にバラバラに散らばっている`
- **本文**:
  > 決済会社の画面を開いてエラーを確認 → CSVを抜いて表計算で集計 → カスタマーサポートが個別に文面を考えて連絡 → 経営報告は別途まとめ直し。
  > **同じ事象を4回処理している** のが現状です。

### Scene 5 — 市場の温度差(0:39 - 0:45 / 6秒)

- **画面**: 海外/日本の2カラム。**実在企業ロゴは出さず、文字だけの中立カード**。
- **見出し**: `海外では対応支援が進み、日本でも運用改善の余地がある`
- **本文**(2カラム):
  - 海外: `失敗した決済への対応を支援する機能やサービスが広がっています`
  - 日本: `CSV・管理画面・メール・報告が分断され、手作業が残りやすい領域`
- **下に小さく**: `この領域を統合的に整理する事業を立ち上げ中。本プロトタイプはその第一歩。`

## Act II — 解決アプローチ(0:45 - 1:00 / 15秒 / 2スライド)

### Scene 6 — 何を作ったか(0:45 - 0:53 / 8秒)

- **画面**: 7つのアイコン(エージェント)が横並びになっているフロー図。
- **見出し**: `7つの AI エージェントが、決済エラー対応を一気通貫で整理`
- **本文**:
  > マスク済み(個人情報を隠した)CSVを入れるだけで、
  > **安全確認 → 分類 → 売上影響 → 顧客対応 → 経営報告 → 再発防止** までを1つのワークフローで処理します。

### Scene 7 — 何をしないか(0:53 - 1:00 / 7秒)

- **画面**: 赤い「行わない」リスト + 緑の「行う」リスト の対比。
- **見出し**: `決済処理には一切触らない設計`
- **行わないこと**(赤): `決済の実行・リトライ・お客様へのメール送信`
- **行うこと**(緑): `データの整理・優先順位の提案・文章の下書きまで`
- **下に小さく**: `→ では、実際の動きをご覧ください`

## Act III — デモ(1:00 - 3:09 / 129秒 / 9画面)

**全シーン共通の作り**:
- 画面下に「これは何の画面か」を1行テロップで毎回表示(初見でも今何を見せられているかわかる)
- クリック前 1 秒の溜め、クリック後 1 秒の余韻
- スクロールは ease-in-out で 3-4 秒かけて

### Scene 8 — ランディング(1:00 - 1:10 / 10秒)

- **操作**: トップページを開く → ゆっくり下にスクロール → 戻る
- **テロップ**: `ランディング画面 — このアプリの位置づけを最初に明示`
- **ハイライト**: 赤いカードで「決済処理・リトライ・お客様送信は行いません」が画面中央に来た瞬間で 2 秒停止

### Scene 9 — アップロード(1:10 - 1:23 / 13秒)

- **操作**: 「サンプルCSVで30秒デモを開始」ボタン → アップロード画面遷移
- **テロップ前半(0-6秒)**: `アップロード画面 — 失敗した決済の一覧(サンプル80件)を読み込む`
- **テロップ後半(6-13秒)**: `安全確認 AI: 個人情報らしき値が混ざっていないか自動チェック → OK`
- **ハイライト**: 右側「Safety Agent」カードの緑チェック

### Scene 10 — タイムライン(1:23 - 1:50 / 27秒)

- **操作**: 「分析を開始」をクリック → タイムライン画面で7エージェントのアイコンが順番に点灯するアニメーションを最後まで見せる(約 18 秒)
- **テロップ ①(0-9秒)**: `7つの AI エージェントが順番に仕事をする様子`
- **テロップ ②(9-18秒)**: `安全確認 → 分類 → 売上計算 → 顧客対応 → 経営報告 → 再発防止`
- **テロップ ③(18-27秒)**: `紫バッジ = Azure OpenAI が文章を生成 / 緑バッジ = ルールで確実に分類・計算`
- **重要**: アニメーションが完全に止まるまでカットしない

### Scene 11 — ダッシュボード(1:50 - 2:05 / 15秒)

- **操作**: 「ダッシュボードを見る →」をクリック
- **テロップ前半(0-7秒)**: `ダッシュボード — 今月、請求できなかった売上の全体像が一目でわかる`
- **テロップ後半(7-15秒)**: `カテゴリ別の内訳: カード期限切れ / 残高不足 / 本人認証失敗 など`
- **ハイライト**: 一番上のKPI数値(今月の損失額)を中央に固定

### Scene 12 — Today's Action Plan(2:05 - 2:20 / 15秒)

- **操作**: タブ「Today's Action Plan」をクリック
- **テロップ前半(0-7秒)**: `今日やることリスト — 推定金額付きの優先タスク`
- **テロップ後半(7-15秒)**: `Finance / CS / Engineering など、想定担当を提案`

### Scene 13 — 経営ブリーフィング(2:20 - 2:35 / 15秒)

- **操作**: タブ「経営ブリーフィング」をクリック → 「Markdownをコピー」ボタンをクリック
- **テロップ前半(0-7秒)**: `経営者向け1ページレポート — Slack や Notion にそのまま貼れる`
- **テロップ後半(7-15秒)**: `数字部分はルールで埋めるので、金額の言い間違いが起こらない`
- **ハイライト**: コピーボタン押下後の「コピーしました」表示

### Scene 14 — 顧客対応 下書き(2:35 - 2:48 / 13秒)

- **操作**: タブ「顧客対応下書き」をクリック → 1枚目の下書きカードにマウスを乗せる
- **テロップ前半(0-7秒)**: `お客様への連絡文の下書き — エラー種別ごとに文面を準備`
- **テロップ後半(7-13秒)**: `**自動送信はしません**。担当者が確認して、自分のメールツールから送る前提`

### Scene 15 — シナリオ切替(2:48 - 3:04 / 16秒)

- **操作**: タブ「Scenario Simulator」→ 3つのボタンを順にクリック(売上回収 → リスク最小 → 顧客体験)
- **テロップ ①(0-5秒)**: `「何を一番大事にするか」で優先順位が変わる`
- **テロップ ②(5-10秒)**: `売上回収重視 — 金額の大きい取引から`
- **テロップ ③(10-16秒)**: `リスク最小化 / 顧客体験重視 でも順序が瞬時に変わる`

### Scene 15.5 — 再発防止(3:04 - 3:09 / 5秒)— 新規追加

- **操作**: タブ「再発防止」を開く(チラ見せ)
- **テロップ**: `来月、同じ問題を減らすための運用改善案も自動で整理`
