# 3分デモ収録 — オペレーター用画面操作台本

ナレーション台本は `docs/demo-script-3min.md` を参照。**こちらは画面操作と時間配分の早見表**です。
読み上げ役と操作役が別の場合に特に有効。

> 想定収録時間: **180 秒(3:00)厳守**
> 録画ツール: macOS QuickTime / OBS / Loom など、画面解像度 1280×800 以上推奨

---

## 0. 事前準備(録画開始の 5 分前)

```bash
# ターミナル A: dev server を確認
cd "/path/to/payment-intelligence-agent"
npm run start &
# → http://localhost:3000 が開けることを確認
```

または **本番URL** (`https://pia-demo-51ff8c.bluebush-37a0c845.japaneast.azurecontainerapps.io`)で本番収録。

- [ ] ブラウザは **シークレットウィンドウ**(拡張機能・ブクマバー・通知非表示)
- [ ] ウィンドウサイズ: **1280×800** 固定(全画面ではない)
- [ ] OSの通知をオフ(集中モード / Do Not Disturb)
- [ ] ブラウザDevToolsは閉じる
- [ ] フォントレンダリングが甘ければズーム 110% に
- [ ] **ランディング `/` を開いておく**(録画開始位置)
- [ ] 別タブで以下を **事前ロード**(初回コンパイル遅延を回避):
  1. `/upload?sample=1`
  2. 任意の `/analyze/<id>/timeline`(一度サンプル分析を走らせて、結果ID をメモ)
  3. `/analyze/<id>/dashboard`, `/action-plan`, `/briefing`, `/scenario`, `/drafts`, `/prevention`

## 2. クリック判定がきつい箇所(注意)

| カット # | 注意点 |
| --- | --- |
| 8 (timeline animation) | 自動アニメは ~5 秒で終わるが、念のため **30 秒** 確保。最後の「ダッシュボードを見る」ボタンを押すまで止めない |
| 22 (draft hover) | コピーしてしまわないようボタンに触れない。本文部分だけホバー |
| 25-27 (scenario click) | クリックの間隔は 4 秒ずつ。連打しない(URL update が間に合わないことがある) |
| 31 (タブ切替) | Cmd+Tab ではなくウィンドウ内タブ切替で。視覚的に「同じウィンドウ内」を維持 |

## 3. NG ワード(うっかり映してはいけない)

- macOS の **メニューバー時計** が個人特定可能になる場合は隠す
- ブラウザの **ブックマークバー / 履歴サジェスト** が映らないように
- DevTools / コンソール
- ターミナルの **実コマンド履歴**(本番URLでない値が出る場合)
- `.env.local` の中身(エディタタブが映らないように)
- 他のSlack/メール通知

## 4. 撮り直しトリガー

以下のどれかが起きたら **必ず撮り直し**:

- カット 8 のアニメーションが途中で止まる(JavaScript エラー)
- カット 18 でコピー表示が出ない(クリップボードAPI失敗)
- カット 25-27 で並び替えが起きない(URL パラメータ反映失敗)
- ページが 1 秒以上の白画面を見せる(コンパイル遅延)→ §0 の事前ロードをやり直す

---

## 5. 編集時のポイント(編集者向け)

- カット 8 のタイムラインアニメは **1.5 倍速でカット可** (3:00 を死守するため)
- カット 31 の Zenn アーキ図表示は **最低 10 秒**(審査員が読む時間)
- BGM を入れる場合は、画面 UI の効果音と被らないボリュームに
- 字幕で **CTA / バッジの値** を 0.5 秒だけ強調すると判定がよりわかりやすい
- 公開設定: **限定公開以上**(Zenn からアクセスできる必要あり)
